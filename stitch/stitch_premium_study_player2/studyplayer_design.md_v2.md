# StudyPlayer 통합 설계 및 구현 가이드 (Premium Scholar v2)

이 문서는 StudyPlayer의 'Deep Focus' 디자인 컨셉을 유지하면서, 사용자 피드백을 반영하여 시각적 인지성을 개선하고 각 핵심 화면의 논리적 구조를 정의합니다.

## 1. 디자인 원칙 및 테마 (Refined Dark)
- **Brightness Adjustment**: 기존의 완벽한 블랙(#000)에서 눈의 피로를 줄이고 요소 간 구분을 명확히 하기 위해 베이스 배경색을 '미세한 딥 차콜'로 상향 조정합니다.
- **Hierarchy through Tonal Layers**: 그림자 대신 톤의 차이(Tonal Elevation)와 1px 보더(Inner Glow)를 사용하여 깊이감을 형성합니다.
- **Interaction Clarity**: 프리미엄 레드는 오답/삭제/긴급 알림에, 스콜라 블루는 진행도/선택/긍정적 액션에 고정 배정합니다.

## 2. 화면별 설계 가이드

### A. 학습(Study) 메인 화면
- **상단**: 학습 진행률(1/23)과 9개의 유틸리티 아이콘(배속, 도움말, 번개, 스왑, 반복, 재생, 편집, 즐겨찾기) 배치.
- **중앙**: 학습 카드(제목, 1회차 뱃지). 정답 확인 시 상세 해설 영역이 슬라이딩 업 또는 확장되는 구조.
- **하단**: 페이지네이션 및 '정답' 버튼. 메인 네비게이션(학습, 목록, 추가, 설정).

### B. 목록(List) 화면
- **필터링**: 검색바와 '전체/도움말/상식/실무/행정' 카테고리 칩.
- **정렬**: '모두/중요/틀린 문제' 토글 버튼.
- **항목**: 각 카드 내에서 '편집, 별표, 삭제' 기능을 아이콘으로 직접 노출.

### C. 문제 추가(Add) 화면
- **입력 모드**: 개별 입력과 대량 불러오기(Bulk) 탭 전환.
- **구성**: 과목 선택 드롭다운, 질문 입력창, 답변 입력창. '암기 노트' 형식의 내용 중심 모드 지원.

### D. 설정(Settings) 화면 (Long-Scroll)
설정은 하나의 긴 흐름을 가지며 세 부분으로 나뉩니다:
1. **학습 모드 & 스타일**: 전체/오답/즐겨찾기 선택, 폰트 선택(Pretendard, Noto Sans 등), 글자 크기 슬라이더.
2. **음성 엔진**: Premium(Edge) vs 기본(Browser) 선택 및 음성 속도 제어, 음성 테스트 버튼.
3. **데이터 & 백업**: 데이터 내보내기/가져오기, 디자인 백업(Auto-timestamp), GitHub 클라우드 동기화(Personal Access Token, Owner, Repo 설정).

### E. PC/모바일 전환 시스템
- **우측 상단 제어**: 'PC/MOBILE' 토글 스위치와 새로고침 버튼.
- **반응형 전략**: PC 모드에서는 모바일 프레임이 중앙에 배치되거나, 넓은 화면을 활용한 2컬럼 레이아웃으로 확장될 수 있는 가변 구조.

## 3. 기술 스택 및 구현 상세
- **CSS**: Tailwind CSS를 기반으로 한 Custom 디자인 토큰 사용.
- **Icons**: Lucide 또는 Heroicons 스타일의 2px Stroke 일관성 유지.
- **Responsive**: Mobile-first 디자인을 기본으로 하되, PC 토글 시 `max-w` 컨테이너와 장치 목업 프레임을 적용.
